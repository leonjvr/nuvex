## ইনস্টলেশন

### পূর্বশর্তসমূহ

| টুল | প্রয়োজনীয় | নোটস |
|-----|-----------|------|
| **Node.js** | >= 22.0.0 | ES মডিউল, `fetch()`, `crypto.subtle`। [ডাউনলোড](https://nodejs.org) |
| **C/C++ টুলচেইন** | শুধু সোর্স বিল্ডের জন্য | `better-sqlite3` এবং `argon2` নেটিভ অ্যাডন কম্পাইল করে |
| **Docker** | >= 24 (ঐচ্ছিক) | শুধুমাত্র Docker ডিপ্লয়মেন্টের জন্য |

Node.js 22 ইনস্টল করুন: Ubuntu/Debian (`curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - && sudo apt-get install -y nodejs`), macOS (`brew install node@22`), Windows (`winget install OpenJS.NodeJS.LTS`)।

C/C++ টুলস ইনস্টল করুন: Ubuntu (`sudo apt-get install -y python3 make g++ build-essential`), macOS (`xcode-select --install`), Windows (`npm install --global windows-build-tools`)।

### বিকল্প A — Docker (প্রস্তাবিত)

```bash
git clone https://github.com/GoetzKohlberg/sidjua.git
cd sidjua
docker compose up -d
```

```bash
# স্বয়ংক্রিয়ভাবে তৈরি API কী দেখুন
docker compose exec sidjua cat /app/.system/api-key

# গভর্ন্যান্স বুটস্ট্র্যাপ করুন
docker compose exec sidjua sidjua apply --verbose

# সিস্টেম স্বাস্থ্য পরীক্ষা
docker compose exec sidjua sidjua selftest
```

**linux/amd64** এবং **linux/arm64** (Raspberry Pi, Apple Silicon) সমর্থন করে।

### বিকল্প B — npm গ্লোবাল ইনস্টল

```bash
npm install -g sidjua
sidjua init          # ইন্টারেক্টিভ ৩-ধাপের সেটআপ
sidjua chat guide    # জিরো-কনফিগ AI গাইড (কোনো API কী লাগবে না)
```

### বিকল্প C — সোর্স বিল্ড

```bash
git clone https://github.com/GoetzKohlberg/sidjua.git
cd sidjua
npm ci && npm run build
node dist/index.js --help
```

## কেন SIDJUA?

আজকের প্রতিটি AI এজেন্ট ফ্রেমওয়ার্ক একই ভাঙা অনুমানের উপর নির্ভর করে: যে আপনি
AI-কে তার নিজের নিয়ম মেনে চলতে বিশ্বাস করতে পারেন।

**প্রম্পট-ভিত্তিক গভর্ন্যান্সের সমস্যা:**

আপনি একটি এজেন্টকে একটি সিস্টেম প্রম্পট দেন যা বলে "কখনো গ্রাহকের PII অ্যাক্সেস করো না।"
এজেন্ট নির্দেশনা পড়ে। এজেন্ট ব্যবহারকারীর বার্তাও পড়ে যেখানে জন স্মিথের পেমেন্ট
ইতিহাস টেনে আনতে বলা হচ্ছে। এজেন্ট — নিজে থেকে — সিদ্ধান্ত নেয় সম্মতি জানাবে কিনা।
এটা গভর্ন্যান্স নয়। এটা একটি জোরালো পরামর্শ।

**SIDJUA আলাদা।**

গভর্ন্যান্স এজেন্টের **বাইরে** থাকে। প্রতিটি অ্যাকশন একটি ৫-ধাপের প্রি-অ্যাকশন
এনফোর্সমেন্ট পাইপলাইনের মধ্য দিয়ে যায় **কার্যকর হওয়ার আগে**। আপনি YAML-এ নিয়ম
সংজ্ঞায়িত করেন। সিস্টেম সেগুলো প্রয়োগ করে। এজেন্ট কখনো সিদ্ধান্ত নিতে পারে না
সেগুলো অনুসরণ করবে কিনা, কারণ চেকটি এজেন্ট কাজ করার আগেই হয়।

এটি আর্কিটেকচার দ্বারা গভর্ন্যান্স — প্রম্পটিং দ্বারা নয়, ফাইন-টিউনিং দ্বারা নয়,
আশার উপর নির্ভর করে নয়।

## এটি কীভাবে কাজ করে

SIDJUA আপনার এজেন্টগুলোকে একটি বাহ্যিক গভর্ন্যান্স স্তরে মোড়া দেয়। এজেন্টের LLM
কল কখনো হয় না যতক্ষণ না প্রস্তাবিত অ্যাকশন একটি ৫-ধাপের এনফোর্সমেন্ট পাইপলাইন
পার না করে:

**ধাপ ১ — নিষিদ্ধ:** অবরুদ্ধ অ্যাকশন অবিলম্বে প্রত্যাখ্যান করা হয়। কোনো LLM কল নেই,
কোনো লগ এন্ট্রি "অনুমোদিত" চিহ্নিত নেই, কোনো দ্বিতীয় সুযোগ নেই। অ্যাকশন নিষিদ্ধ
তালিকায় থাকলে, এটি এখানেই থামে।

**ধাপ ২ — অনুমোদন:** মানব সাইন-অফ প্রয়োজন এমন অ্যাকশনগুলো কার্যকরের আগে
অনুমোদনের জন্য ধরে রাখা হয়। এজেন্ট অপেক্ষা করে। মানুষ সিদ্ধান্ত নেয়।

**ধাপ ৩ — বাজেট:** প্রতিটি টাস্ক রিয়েল-টাইম ব্যয় সীমার বিপরীতে চলে। প্রতি-টাস্ক
এবং প্রতি-এজেন্ট বাজেট প্রয়োগ করা হয়। সীমায় পৌঁছালে, টাস্কটি বাতিল করা হয় —
পতাকাঙ্কিত নয়, পর্যালোচনার জন্য লগ করা নয়, *বাতিল করা হয়*।

**ধাপ ৪ — শ্রেণীবিভাগ:** বিভাগের সীমানা অতিক্রমকারী ডেটা শ্রেণীবিভাগ নিয়মের বিপরীতে
পরীক্ষা করা হয়। টায়ার-২ এজেন্ট SECRET ডেটা অ্যাক্সেস করতে পারে না। বিভাগ A-এর
এজেন্ট বিভাগ B-এর গোপনীয়তা পড়তে পারে না।

**ধাপ ৫ — নীতি:** কাস্টম সাংগঠনিক নিয়ম, কাঠামোগতভাবে প্রয়োগ করা হয়। API কল
ফ্রিকোয়েন্সি সীমা, আউটপুট টোকেন ক্যাপ, টাইম-উইন্ডো বিধিনিষেধ।

সম্পূর্ণ পাইপলাইন যেকোনো অ্যাকশন কার্যকর হওয়ার আগে চলে। গভর্ন্যান্স-সমালোচনামূলক
অপারেশনগুলোর জন্য কোনো "লগ করুন এবং পরে পর্যালোচনা করুন" মোড নেই।

### একক কনফিগারেশন ফাইল

আপনার সম্পূর্ণ এজেন্ট অর্গানাইজেশন একটি `divisions.yaml`-এ থাকে:

```yaml
divisions:
  - name: engineering
    agents:
      - name: research-agent
        provider: anthropic
        model: claude-haiku-4-5-20251001
        tier: 2
        budget:
          per_task_usd: 0.50
          per_month_usd: 50.00
    governance:
      rules:
        - no_external_api_calls: true
        - max_tokens_per_response: 4096
        - require_human_approval: [delete, send_email]
```

`sidjua apply` এই ফাইলটি পড়ে এবং সম্পূর্ণ এজেন্ট অবকাঠামো প্রভিশন করে:
এজেন্ট, বিভাগ, RBAC, রাউটিং, অডিট টেবিল, গোপনীয়তা পাথ এবং গভর্ন্যান্স নিয়ম
— ১০টি পুনরুৎপাদনযোগ্য ধাপে।

### এজেন্ট আর্কিটেকচার

এজেন্টগুলো **বিভাগ** (কার্যকরী গ্রুপ) এবং **টায়ার** (বিশ্বাস স্তর)-এ সংগঠিত।
টায়ার ১ এজেন্টগুলোর তাদের গভর্ন্যান্স এনভেলপের মধ্যে সম্পূর্ণ স্বায়ত্তশাসন রয়েছে।
টায়ার ২ এজেন্টগুলো সংবেদনশীল অপারেশনের জন্য অনুমোদন প্রয়োজন। টায়ার ৩ এজেন্টগুলো
সম্পূর্ণরূপে তত্ত্বাবধানে থাকে। টায়ার সিস্টেম কাঠামোগতভাবে প্রয়োগ করা হয় —
একটি এজেন্ট নিজেকে উন্নীত করতে পারে না।

## আর্কিটেকচার সীমাবদ্ধতা

SIDJUA এই সীমাবদ্ধতাগুলো আর্কিটেকচার স্তরে প্রয়োগ করে — সেগুলো এজেন্টদের দ্বারা
নিষ্ক্রিয়, বাইপাস বা ওভাররাইড করা যায় না:

1. **গভর্ন্যান্স বাহ্যিক**: গভর্ন্যান্স স্তর এজেন্টকে মোড়া দেয়। এজেন্টের গভর্ন্যান্স
   কোডে কোনো অ্যাক্সেস নেই, নিয়ম পরিবর্তন করতে পারে না, এবং গভর্ন্যান্স উপস্থিত
   কিনা তা সনাক্ত করতে পারে না।

2. **প্রি-অ্যাকশন, পোস্ট-অ্যাকশন নয়**: প্রতিটি অ্যাকশন কার্যকরের আগে পরীক্ষা করা হয়।
   গভর্ন্যান্স-সমালোচনামূলক অপারেশনগুলোর জন্য কোনো "লগ করুন এবং পরে পর্যালোচনা করুন"
   মোড নেই।

3. **কাঠামোগত প্রয়োগ**: নিয়মগুলো কোড পাথ দ্বারা প্রয়োগ করা হয়, প্রম্পট বা মডেল
   নির্দেশনা দ্বারা নয়। একটি এজেন্ট গভর্ন্যান্স থেকে "জেলব্রেক" করতে পারে না
   কারণ গভর্ন্যান্স মডেলের নির্দেশনা হিসেবে প্রয়োগ করা হয় না।

4. **অডিট অপরিবর্তনীয়তা**: রাইট-অ্যাহেড লগ (WAL) অখণ্ডতা যাচাইয়ের সাথে শুধুমাত্র
   অ্যাপেন্ড করা যায়। টেম্পার করা এন্ট্রিগুলো সনাক্ত এবং বাদ দেওয়া হয়।

5. **বিভাগ বিচ্ছিন্নতা**: বিভিন্ন বিভাগের এজেন্টগুলো একে অপরের ডেটা, গোপনীয়তা
   বা যোগাযোগ চ্যানেল অ্যাক্সেস করতে পারে না।

## ফিচারসমূহ

### গভর্ন্যান্স ও কমপ্লায়েন্স

**প্রি-অ্যাকশন পাইপলাইন (স্টেজ ০)** প্রতিটি এজেন্ট অ্যাকশনের আগে চলে: নিষিদ্ধ
চেক → মানব অনুমোদন → বাজেট এনফোর্সমেন্ট → ডেটা শ্রেণীবিভাগ → কাস্টম নীতি।
পাঁচটি ধাপই কাঠামোগত — সেগুলো কোডে কার্যকর হয়, এজেন্টের প্রম্পটে নয়।

**বাধ্যতামূলক বেসলাইন নিয়মাবলী** প্রতিটি ইনস্টলেশনের সাথে আসে: ১০টি গভর্ন্যান্স
নিয়ম (`SYS-SEC-001` থেকে `SYS-GOV-002`) যা ব্যবহারকারীর কনফিগারেশন দ্বারা সরানো
বা দুর্বল করা যায় না। কাস্টম নিয়মগুলো বেসলাইন প্রসারিত করে; সেগুলো ওভাররাইড
করতে পারে না।

**EU AI Act কমপ্লায়েন্স** — অডিট ট্রেইল, শ্রেণীবিভাগ কাঠামো এবং অনুমোদন
ওয়ার্কফ্লো সরাসরি আর্টিকেল ৯, ১২ এবং ১৭ প্রয়োজনীয়তার সাথে মানচিত্র তৈরি করে।
আগস্ট ২০২৬ কমপ্লায়েন্স ডেডলাইন পণ্যের রোডম্যাপে নির্মিত।

**কমপ্লায়েন্স রিপোর্টিং** `sidjua audit report/violations/agents/export`-এর মাধ্যমে:
কমপ্লায়েন্স স্কোর, প্রতি-এজেন্ট ট্রাস্ট স্কোর, লঙ্ঘনের ইতিহাস, বাহ্যিক অডিটর বা SIEM
ইন্টিগ্রেশনের জন্য CSV/JSON এক্সপোর্ট।

**রাইট-অ্যাহেড লগ (WAL)** অখণ্ডতা যাচাইয়ের সাথে: প্রতিটি গভর্ন্যান্স সিদ্ধান্ত
কার্যকরের আগে একটি শুধুমাত্র-অ্যাপেন্ড লগে লেখা হয়। টেম্পার করা এন্ট্রিগুলো পড়ার
সময় সনাক্ত করা হয়। `sidjua memory recover` পুনঃযাচাই করে এবং মেরামত করে।

### যোগাযোগ

এজেন্টগুলো শুধু API কলের প্রতিক্রিয়া জানায় না — তারা বাস্তব যোগাযোগ চ্যানেলে
অংশগ্রহণ করে।

**দ্বিমুখী ইমেইল** (`sidjua email status/test/threads`): এজেন্টগুলো IMAP পোলিংয়ের
মাধ্যমে ইমেইল পায় এবং SMTP-এর মাধ্যমে উত্তর দেয়। In-Reply-To হেডারের মাধ্যমে
থ্রেড ম্যাপিং কথোপকথন সংগত রাখে। প্রেরক হোয়াইটলিস্টিং, বডি সাইজ সীমা এবং
HTML স্ট্রিপিং এজেন্ট পাইপলাইনকে দূষিত ইনপুট থেকে রক্ষা করে।

**Discord গেটওয়ে বট**: `sidjua module install discord`-এর মাধ্যমে সম্পূর্ণ স্ল্যাশ-কমান্ড
ইন্টারফেস। এজেন্টগুলো Discord বার্তার প্রতিক্রিয়া জানায়, কথোপকথন থ্রেড বজায়
রাখে এবং সক্রিয় বিজ্ঞপ্তি পাঠায়।

**Telegram ইন্টিগ্রেশন**: Telegram বটের মাধ্যমে এজেন্ট সতর্কতা এবং বিজ্ঞপ্তি।
মাল্টি-চ্যানেল অ্যাডাপ্টার প্যাটার্ন সমান্তরালে Telegram, Discord, ntfy এবং ইমেইল
সমর্থন করে।

### অপারেশন

প্রোডাকশনে **একক Docker কমান্ড**:

```bash
docker run -p 4200:4200 sidjua/sidjua:latest
```

API কী প্রথম স্টার্টে স্বয়ংক্রিয়ভাবে তৈরি হয় এবং কন্টেইনার লগে প্রিন্ট করা হয়।
কোনো পরিবেশ ভেরিয়েবল প্রয়োজন নেই। কোনো কনফিগারেশন প্রয়োজন নেই। কোনো ডেটাবেস
সার্ভার প্রয়োজন নেই — SIDJUA SQLite ব্যবহার করে, প্রতি এজেন্টের জন্য একটি ডেটাবেস ফাইল।

**CLI ম্যানেজমেন্ট** — একটি একক বাইনারি থেকে সম্পূর্ণ জীবনচক্র:

```bash
sidjua init                      # ইন্টারেক্টিভ ওয়ার্কস্পেস সেটআপ (৩ ধাপ)
sidjua apply                     # divisions.yaml থেকে প্রভিশন করুন
sidjua agent create/list/stop    # এজেন্ট জীবনচক্র
sidjua run "task..." --wait      # গভর্ন্যান্স এনফোর্সমেন্ট সহ টাস্ক জমা দিন
sidjua audit report              # কমপ্লায়েন্স রিপোর্ট
sidjua costs                     # বিভাগ/এজেন্ট অনুযায়ী ব্যয় বিভাজন
sidjua backup create/restore     # HMAC-স্বাক্ষরিত ব্যাকআপ ব্যবস্থাপনা
sidjua update                    # স্বয়ংক্রিয় প্রি-ব্যাকআপ সহ ভার্সন আপডেট
sidjua rollback                  # পূর্ববর্তী ভার্সনে ১-ক্লিক পুনরুদ্ধার
sidjua email status/test         # ইমেইল চ্যানেল ব্যবস্থাপনা
sidjua secret set/get/rotate     # এনক্রিপ্টেড গোপনীয়তা ব্যবস্থাপনা
sidjua memory import/search      # সিমান্টিক জ্ঞান পাইপলাইন
sidjua selftest                  # সিস্টেম স্বাস্থ্য পরীক্ষা (৭ বিভাগ, ০-১০০ স্কোর)
```

**সিমান্টিক মেমরি** — কথোপকথন এবং ডকুমেন্ট আমদানি করুন (`sidjua memory import
~/exports/claude-chats.zip`), ভেক্টর + BM25 হাইব্রিড র‍্যাঙ্কিং দিয়ে অনুসন্ধান করুন।
Cloudflare Workers AI এম্বেডিং (বিনামূল্যে, জিরো-কনফিগ) এবং OpenAI বড় এম্বেডিং
(বড় জ্ঞান বেসের জন্য উচ্চ মান) সমর্থন করে।

**অ্যাডাপ্টিভ চাংকিং** — মেমরি পাইপলাইন স্বয়ংক্রিয়ভাবে প্রতিটি এম্বেডিং মডেলের
টোকেন সীমার মধ্যে থাকতে চাংক সাইজ সামঞ্জস্য করে।

**জিরো-কনফিগ গাইড** — `sidjua chat guide` কোনো API কী ছাড়াই একটি ইন্টারেক্টিভ AI
সহকারী চালু করে, SIDJUA প্রক্সির মাধ্যমে Cloudflare Workers AI দ্বারা চালিত।
এজেন্ট সেটআপ করতে, গভর্ন্যান্স কনফিগার করতে, বা অডিট লগে কী ঘটেছে তা বুঝতে
জিজ্ঞেস করুন।

**এয়ার-গ্যাপ ডিপ্লয়মেন্ট** — Ollama বা যেকোনো OpenAI-সামঞ্জস্যপূর্ণ এন্ডপয়েন্টের
মাধ্যমে স্থানীয় LLM ব্যবহার করে ইন্টারনেট থেকে সম্পূর্ণ বিচ্ছিন্নভাবে চালান।
ডিফল্টরূপে কোনো টেলিমেট্রি নেই। সম্পূর্ণ PII রিডাকশন সহ ঐচ্ছিক অপ্ট-ইন ক্র্যাশ রিপোর্টিং।

### নিরাপত্তা

**স্যান্ডবক্স বিচ্ছিন্নতা** — এজেন্ট স্কিলগুলো bubblewrap (Linux ব্যবহারকারী
নেমস্পেস)-এর মাধ্যমে OS-স্তরের প্রক্রিয়া বিচ্ছিন্নতার মধ্যে চলে। শূন্য অতিরিক্ত
RAM ওভারহেড। প্লাগযোগ্য `SandboxProvider` ইন্টারফেস: ডেভেলপমেন্টের জন্য `none`,
প্রোডাকশনের জন্য `bubblewrap`।

**গোপনীয়তা ব্যবস্থাপনা** — RBAC সহ এনক্রিপ্টেড গোপনীয়তা স্টোর (`sidjua secret
set/get/list/delete/rotate/namespaces`)। কোনো বাহ্যিক ভল্ট প্রয়োজন নেই।

**নিরাপত্তা-প্রথম বিল্ড** — বিস্তৃত অভ্যন্তরীণ টেস্ট স্যুট এবং ২ জন বাহ্যিক কোড
অডিটর (DeepSeek V3 এবং xAI Grok) দ্বারা স্বাধীন যাচাইকরণ। প্রতিটি API সারফেসে
নিরাপত্তা হেডার, CSRF সুরক্ষা, রেট লিমিটিং এবং ইনপুট স্যানিটাইজেশন। সর্বত্র
প্যারামিটারযুক্ত কোয়েরি সহ SQL ইনজেকশন প্রতিরোধ।

**ব্যাকআপ অখণ্ডতা** — HMAC-স্বাক্ষরিত ব্যাকআপ আর্কাইভ zip-slip সুরক্ষা, zip বোমা
প্রতিরোধ এবং পুনরুদ্ধারে ম্যানিফেস্ট চেকসাম যাচাইকরণ সহ।

# কী আমদানি হবে তার প্রিভিউ দেখুন — কোনো পরিবর্তন করা হয় না
sidjua import openclaw --dry-run

sidjua import openclaw --skills

স্বয়ংক্রিয়ভাবে গভর্ন্যান্স, অডিট ট্রেইল এবং বাজেট নিয়ন্ত্রণ যোগ করে।

## কনফিগারেশন রেফারেন্স

শুরু করার জন্য একটি ন্যূনতম `divisions.yaml`:

```yaml
organization:
  name: "my-org"
  tier: 1

divisions:
  - name: operations
    tier: 2
    agents:
      - name: ops-agent
        provider: anthropic
        model: claude-haiku-4-5-20251001
        division: operations
        budget:
          per_task_usd: 0.25
          per_month_usd: 25.00

governance:
  stage0:
    enabled: true
    forbidden_actions:
      - delete_database
      - exfiltrate_data
    classification:
      default_level: INTERNAL
      max_agent_level: CONFIDENTIAL
```

`sidjua apply` এই ফাইল থেকে সম্পূর্ণ অবকাঠামো প্রভিশন করে। পরিবর্তনের পরে
আবার চালান — এটি আইডেম্পোটেন্ট।

সমস্ত ১০টি প্রভিশনিং ধাপের সম্পূর্ণ স্পেসিফিকেশনের জন্য
[docs/SIDJUA-APPLY-TECH-SPEC-V1.md](docs/SIDJUA-APPLY-TECH-SPEC-V1.md) দেখুন।

## REST API

SIDJUA REST API ড্যাশবোর্ডের মতো একই পোর্টে চলে:

```bash
sidjua server start --port 4200 --api-key $SIDJUA_API_KEY
```

মূল এন্ডপয়েন্ট:

```
GET  /api/v1/health          # পাবলিক স্বাস্থ্য চেক (কোনো অথ নেই)
GET  /api/v1/info            # সিস্টেম মেটাডেটা (প্রমাণীকৃত)
POST /api/v1/execute/run     # টাস্ক জমা দিন
GET  /api/v1/execute/:id/status  # টাস্কের অবস্থা
GET  /api/v1/execute/:id/result  # টাস্কের ফলাফল
GET  /api/v1/events          # SSE ইভেন্ট স্ট্রিম
GET  /api/v1/audit/report    # কমপ্লায়েন্স রিপোর্ট
```

`/health` ছাড়া সমস্ত এন্ডপয়েন্ট Bearer প্রমাণীকরণ প্রয়োজন। একটি কী তৈরি করুন:

```bash
sidjua api-key generate
```

## Docker Compose

```yaml
services:
  sidjua:
    image: sidjua/sidjua:latest
    ports:
      - "4200:4200"
    volumes:
      - sidjua-data:/data
    restart: unless-stopped

volumes:
  sidjua-data:
```

অথবা অন্তর্ভুক্ত `docker-compose.yml` ব্যবহার করুন যা কনফিগ, লগ এবং এজেন্ট
ওয়ার্কস্পেসের জন্য নামযুক্ত ভলিউম যোগ করে, এবং সিমান্টিক সার্চের জন্য একটি
ঐচ্ছিক Qdrant সার্ভিস:

```bash
docker compose up -d
docker compose exec sidjua sidjua init
docker compose exec sidjua sidjua apply
```

## প্রভাইডারসমূহ

SIDJUA লক-ইন ছাড়াই যেকোনো LLM প্রভাইডারের সাথে সংযুক্ত হয়:

| প্রভাইডার | মডেল | API কী |
|-----------|------|--------|
| Anthropic | Claude Sonnet 4.6, Haiku 4.5, Opus 4.6 | `ANTHROPIC_API_KEY` |
| Google | Gemini 2.0, Gemini 1.5 Pro | `GOOGLE_API_KEY` |
| Groq | Llama 3.x, Mixtral | `GROQ_API_KEY` (বিনামূল্যের টায়ার) |
| Cloudflare Workers AI | Llama, Mistral, Phi | `CF_ACCOUNT_ID` + `CF_TOKEN` |
| Ollama | যেকোনো স্থানীয় মডেল | কোনো কী নেই (স্থানীয়) |
| OpenAI-সামঞ্জস্যপূর্ণ | যেকোনো এন্ডপয়েন্ট | কাস্টম URL + কী |

```bash
# একটি প্রভাইডার কী যোগ করুন
sidjua key set groq gsk_...

# উপলব্ধ প্রভাইডার এবং মডেল তালিকা করুন
sidjua provider list
```
